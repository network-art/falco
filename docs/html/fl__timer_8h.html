Detailed Description

Timer Management.

Applications can use this module to create, start (arm), stop (disarm) and delete timers. Methods (or callback routines) can be associated with a timer. They are invoked when the timer fires (i.e. upon timeout).

This module uses the timerfd infrastructure in linux. Underneath the APIs, timerfd_create() is to used to create a timer. timerfd_settime() is used to arm or disarm the timer. When a timer is deleted, the file descriptor associated with the timer is closed.

#include <sys/queue.h>
#include <sys/timerfd.h>
#include "falco/fl_task.h"

Create a new falco timer.

This function creates a new falco timer object. timerfd_create() is used to create the timer, and the returned file descriptor is stored in timerfd.

Parameters
[in]task(Optional) Task to which this timer needs to be associated
[in]fire_whenOne shot timer expiration value
[in]fire_intervalPeriodic interval value
[in]timer_methodMethod (or callback routine) invoked when the timer fires (reaches expiration)
[in]timer_nameString of length not exceeding FL_TIMER_NAME_MAX_LEN
[in]app_dataApplication data (or context) that needs to be presented to the application when the timeout method is called.
Returns
On success, a pointer to a falco timer object is returned. Otherwise, NULL is returned.

Dispatch all timers which have expirations.

This function iterates through all the timers, checks if the timer has expirations and invokes the corresponding application methods (with the data or context presented by the application earlier).

Parameters
[in,out]nfdsNumber of file descriptors that need to be read. It is decremented by the number of timers that were dispatched in this dispatch run.
[in,out]rfdsSet of file descriptors (associated with the timers).
